## Loading the Payloads

1. Open your Rubber Ducky and place the stickers so that, when reassembled, they sit on top of the button.  
2. Raise your hand if you need help opening it; we have pry tools to open it safely for you.  
3. In a browser, go to [https://payloadstudio.com/community/](https://payloadstudio.com/community/).  
4. Press the button on the Ducky to put it into **Storage Mode**.  
5. Open this repository, find the relevant `payload.txt`, and copy its contents into the PayloadStudio editor.  
6. Click **Generate Payload** at the bottom-left of PayloadStudio.  
7. Download the generated payload (and source if you want a copy).  
8. **Cut** and paste the downloaded payload file onto your Ducky.  
    i. If you **copy** instead of cut, make sure to delete the original file so the final filename on the Ducky is exactly `inject.bin` (names like `inject (1).bin` will **not** work).  
9. Repeat steps 3–9 for each example you want to load.

## Writing script for Rubber Ducky
You'll need to change the language on the https://payloadstudio.com/community/ website when compiliing.
1. Go to https://payloadstudio.com/community/
2. Select Settings
3. Select Compiler Settings
4. Change language to gb.json
